Two coexisting styling approaches:
- **CSS Modules** (`*.module.css`) for layout shells and the conversation/property components (e.g. `dashboard.module.css`, `ConversationList.module.css`, `IconSidebar.module.css`, `NewPropertyForm`, `PropertyProfile` (the property detail component, was `PropertyEditor` before the 2026-07 redesign), `MessageThread`). These reference design-token CSS variables. The Properties section (list page `properties.module.css` + `PropertyProfile`) is entirely CSS Modules — keep new work there in module.css for cohesion.
- **Tailwind v4 utility classes** for newer dashboard pages AND their extracted data-display components (overview, conversations split-pane, maintenance). Arbitrary-value utilities like `text-[#344767]` and `shadow-[0_4px_24px_rgba(52,71,103,0.10)]` map to the token palette.

**Which one for a NEW component?** Follow the closest precedent: newer dashboard/data-display components under `components/` use Tailwind inline, not module.css — e.g. `components/maintenance/TicketList.tsx` and `components/dashboard/*` (TrendChart, PropertySummaryList) are all Tailwind. Reserve `.module.css` for chrome/layout shells and the older conversation/property form components. (An orchestrator once suggested module.css for a new extracted chart component; the actual TicketList precedent is Tailwind, and dynamic SVG geometry must be inline anyway, so Tailwind was the right call.)

**Dashboard stat tiles / gradients:** brand gradient is `linear-gradient(135deg,#42a5f5,#1565c0)`; a green metric gradient `linear-gradient(135deg,#66bb6a,#0f9d58)` pairs with the existing green messages accent. In Tailwind v4 apply gradients via the arbitrary `[background:linear-gradient(...)]` utility (underscores for spaces). Per-metric accents: tickets blue `#1565c0`, messages green `#0f9d58`, escalations red `#d93025`.

**Design tokens live in `app/globals.css` `:root` and are the single source of truth for color.** As of the 2026-07 palette consolidation, EVERY `*.module.css` references `var(--color-…)` / `var(--shadow-…)` / `var(--radius-…)` — do NOT introduce raw hex/rgba for blue/white/grey fills, text, or borders; reuse a token. Key values:
- Brand: `--color-brand` #1976d2, `--color-brand-dark` #1565c0 (primary accent, active states, badges), `--color-brand-light` #42a5f5, `--color-brand-gradient` (135deg light→dark, used for avatars/buttons/outbound bubbles), `--color-brand-border` rgba(25,118,210,0.35) (dashed/outline button borders — replaced ~8 near-dup .25/.3/.35/.4 tints)
- Backgrounds: `--color-bg-base` #ffffff (page canvas), `--color-bg-surface` #ffffff (cards), `--color-bg-elevated` #e8f0fe (active/selected highlight, also brand-tint hover), `--color-bg-hover` #f5f8ff
- Borders: `--color-border` rgba(52,71,103,0.10), `--color-border-subtle` rgba(52,71,103,0.05); inputs `--color-input-bg` #f8fafc, `--color-input-border` rgba(52,71,103,0.18), `--color-input-border-focus` #1976d2
- Text: `--color-text-primary` #1e293b, `--color-text-secondary` #7b809a, `--color-text-muted` #b0b7c3. White text on brand/gradient = `--color-btn-text` #ffffff.
- Semantic states: `--color-error` #ef5350 (SOFT red — delete affordances only) + `--color-error-bg-hover` rgba(239,83,80,0.08) (delete/remove hover bg), `--color-danger` #d93025 + `--color-danger-bg` #fce8e6 (STRONG red — escalation/alert badges & error boxes), `--color-success` #4caf50 + `--color-success-bg` #e6f4ea (green "all clear" badges), `--color-warning` #9a3412 + `--color-warning-bg` #ffedd5 (amber "needs attention"/maintenance badges)
- Overlays (modal/drawer scrims): `--color-overlay` rgba(15,23,42,0.5), `--color-overlay-subtle` rgba(15,23,42,0.3)
- On-brand: `--color-border-on-brand` rgba(255,255,255,0.4) (translucent divider/spinner track over brand surfaces, e.g. auth + message-input spinners)
- Accents (one-offs): `--color-avatar-staff-start` #334155 (start stop of staff-avatar slate gradient in MessageThread), `--color-highlight` #fff176 (search-match highlight yellow)
- Shadows: `--shadow-hairline` (1px header underline), `--shadow-xs`, `--shadow-sm`, `--shadow-card`, `--shadow-card-hover`, `--shadow-sidebar`, `--shadow-drawer` (right-edge slide-out), `--shadow-button` (brand glow), `--shadow-button-hover` (lifted brand button — consolidated 3 near-dup 0 6px16px/0 4px10px brand glows), `--shadow-tooltip` (dark chrome tooltip), `--shadow-focus` (0 0 0 3px brand @.12, input focus ring); radius `--radius-sm/md/lg/xl/bubble/pill`
- Sidebar chrome (self-contained dark surface): `--color-sidebar-bg` #1a2332, `--color-sidebar-border` rgba(255,255,255,0.04), `--color-sidebar-hover-bg` rgba(255,255,255,0.07), `--color-sidebar-hover-text` #cbd5e1, `--color-sidebar-tooltip-bg` #0f172a, `--color-sidebar-tooltip-text` #e2e8f0

**Grey collapse rule (narrow palette):** the app deliberately uses only 3 greys — primary/secondary/muted. When you meet a stray slate, map it: `#94a3b8`→`--color-text-muted`; `#64748b`/`#475569`→`--color-text-secondary`; `#344767`→`--color-text-primary`. Don't add new grey shades.

**Escalation is RED app-wide:** escalated-conversation badges and the conversation-list escalated dot both use `--color-danger` (the dot was formerly amber #f59e0b — unified to red).

**Dashboard chrome:** the icon sidebar (`components/sidebar/IconSidebar.module.css`) is a self-contained DARK rail (~68px). Its dark-neutral values are NOT part of the light token set but are now their own `--color-sidebar-*` token group (see token list above) rather than raw hardcodes; brand-blue accents use `--color-brand*`/`--shadow-button`/`--shadow-button-hover`. Brand logo asset is `/tenatimmy_solo.png`. Nav routes: `/dashboard` (Overview, exact), `/dashboard/conversations`, `/dashboard/properties`, `/dashboard/maintenance` (prefix match).

**Maintenance is now a Kanban board** (`components/maintenance/TicketList.tsx`, default export `TicketBoard`, all Tailwind-arbitrary). 4 columns New/In Progress/In Review/Resolved mapping from ticket status (resolved+closed → Resolved). The page (`app/dashboard/maintenance/page.tsx`) is a thin server shell: fetches flat tickets, renders "Maintenance" title + Requests Board/Recurring tabs (Recurring is an inactive placeholder), passes flat array to the client board which does search filtering + column grouping + `router.refresh()` after POST/PATCH. Cards show `#${id.slice(0,6).toUpperCase()}` ref, category title, property subtitle, priority dot-badge, and an assignee avatar (first letter, deterministic color hashed from name via a fixed 6-color palette; null → dashed "Unassigned"). Kebab menu PATCHes status; detail modal edits status+assigned_to.
**Priority dot-badge** (severity → label, in TicketBoard): mild→"Low" green dot `#4caf50`/text `#0f9d58`; moderate→"Normal" blue dot `#42a5f5`/text `#1565c0`; severe→"High" red dot `#ef5350`/text `#d93025`; null → treated as Normal. Replaced the old pill-style `SEVERITY_STYLES` (yellow/orange/red) for the board.

**Maintenance status pill** (property list card + detail, from open ticket count across `units[].tickets`; "open" = status in `['open','in_progress','in_review']` — see [[supabase-data]]): "All clear" now uses `var(--color-success-bg)`/`var(--color-success)`; "Needs attention"/"N open ticket(s)" uses `var(--color-warning-bg)`/`var(--color-warning)`.

**Token consolidation is COMPLETE (2026-07-08 final pass).** A grep of all `*.module.css` for hex/rgba now returns exactly ONE match: `components/properties/NewPropertyForm.module.css:39` `box-shadow: 0 20px 60px rgba(15,23,42,0.25)` — the new-property modal's large drop shadow. It was NOT in the consolidation mapping (mapping had 23 items for a stated "24 values", off by one) and was deliberately left as-is rather than inventing an untracked `--shadow-modal` token without sign-off. If you touch this, the clean move is to add a `--shadow-modal` token to globals.css and reference it. Everything else (sidebar chrome, scrims, hairline/xs/sm/drawer/button-hover/tooltip shadows, error-hover tint, highlight, staff-avatar gradient stop) is now tokenized.

Safe to render in both server and client components. Dynamic Supabase Storage photos use a plain `<img>` with `// eslint-disable-next-line @next/next/no-img-element` (no `remotePatterns` configured in next.config; matches TicketList).
