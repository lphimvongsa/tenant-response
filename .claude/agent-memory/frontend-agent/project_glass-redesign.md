The staff dashboard is being restyled from the flat light-lavender look to a **dark purple gradient background with glassmorphism panels**, rolled out **one page at a time**. See [[styling-conventions]] for the underlying light token system that untouched pages still use.

**Rollout state (as of 2026-07-22):** Migrated so far: **Overview page** (`app/dashboard/page.tsx` + `DashboardTrends`, `TrendChart`, `ResponseChart`, `ConversationSummaryList`) — pass 1; the **Settings page** (`app/dashboard/settings/page.tsx` + all of `components/settings/*`); the **Maintenance page** (`app/dashboard/maintenance/page.tsx` + `loading.tsx`, `components/maintenance/TicketList.tsx`, `MobileTicketList.tsx`); the **Conversations page** (`app/dashboard/conversations/**` + all `components/conversations/*`); and the **Properties page** (`app/dashboard/properties/**` + `components/properties/PropertyProfile(.module.css)`, `NewPropertyForm.module.css`, `properties.module.css`, `PropertyPhotoPlaceholder.tsx`, both `loading.tsx`). **All five dashboard pages are now migrated** — the end-state cleanup is now unblocked (drop `.main`'s light background in `app/dashboard/dashboard.module.css` so the shell gradient shows globally, and revisit glassing the shared sidebar/tab-bar chrome), but that shared-shell pass was scoped separately.
- **Why:** deliberate incremental redesign so each page can be reviewed on its own. (Settings, Maintenance, Conversations, and Properties were done as concurrent "pass 2" passes and diverged slightly — see the divergence notes below.)
- **How to apply:** when told to migrate one of the remaining pages, reuse the glass tokens/utilities below; don't re-invent them. Recolor any dark-ink text/`--color-bg-surface` cards to the on-glass equivalents.

**Conversations page migration specifics (reference for split-view / chat-style pages; it uses CSS Modules, not Tailwind arbitrary values, per [[styling-conventions]]):**
- **Where the gradient is painted:** `ConversationsSplitView.module.css` `.container` (the route's root NON-scrolling fill container — the split-view analogue of Overview's root scroll div). List pane = frosted glass (`--glass-bg` + `backdrop-filter`); detail pane transparent so the gradient shows through the header/thread/input seamlessly.
- **Message bubbles (recolored per the "outbound stays a solid accent" instruction):** inbound = glass chip (`--glass-bg-strong` + `--glass-border` + `--color-on-glass` text); outbound = **solid `--color-lavender-400`** + `--color-lavender-50` text. Outbound avatar also solid lavender-400; inbound avatar = glass chip. Search `.highlight` = solid `#ffd166` + ink text (pops inside either bubble type).
- **Reused conventions verbatim:** unread badge + "Mass text" CTA = solid `--color-lavender-300` pill + `--color-ink` text; primary CTAs (Save contact / Send / modal primary) = `--color-lavender-400`; danger text `#ffb4b4`; `<select>` option popup pinned dark (`--color-ink` bg / `--color-on-glass` text). Conversation-list avatars = deterministic *translucent tinted* chips (lavender/mint/amber/rose/sky ~0.24α) with a bright label — replaced the old solid ink/success/warning/danger palette.
- **Menus / modals / drawers glass treatment (status + kebab dropdowns, MassTextModal, EditContactPanel drawer):** `--glass-bg-strong` + `backdrop-filter: blur(--glass-blur)` + `--glass-shadow`. **TOKEN GAP flagged to user:** there's no dedicated *opaque elevated surface* token — glass-bg-strong (0.14 white) over the gradient is legible but faint for dense popover/form surfaces; add one rather than hardcoding if they want denser menus.
- **Dead files (don't treat as live):** `components/conversations/ConversationHeader.tsx` + `ConversationSearchBar.tsx` are unused (only self-referenced; the live header/search is inlined in `ConversationView.tsx`). They used non-resolving semantic Tailwind classes (`bg-bg-surface`, `text-text-primary`…) — migrated them to arbitrary-value glass anyway. **Two `conversation.module.css` files exist:** the in-scope `app/dashboard/conversations/[conversationId]/conversation.module.css` is dead (detail view uses `ConversationView.module.css`); `app/dashboard/[conversationId]/conversation.module.css` is a **separate legacy route still on the light theme**, outside the conversations-page scope.

**Properties page migration specifics (CSS-Module-based, per [[styling-conventions]]):**
- **Where the gradient is painted:** `properties.module.css` `.page` (list scroll container) + its `.empty` error/empty branch (renders outside `.page`); `PropertyProfile.module.css` `.page` (detail scroll container). In a CSS Module, replicate the glass **tokens** directly (`background: var(--glass-bg); border ... var(--glass-border); box-shadow: var(--glass-shadow); backdrop-filter: blur(var(--glass-blur))`) — don't try to compose the global `.glass-panel` class. Panels (property card / profile card / units section) get backdrop-blur; nested chips (stat tiles, unit cards, tenant rows, tenant-picker options) use `--glass-bg-strong` + `--glass-border` with NO blur so it isn't stacked.
- **Reused the pass-2 form conventions verbatim:** primary CTAs (New/Save/Create/Add) = `--color-lavender-300` + `--color-ink` text + `--color-lavender-200` hover + `--glass-shadow`; inputs = `--glass-bg-strong` fill + `--glass-border-strong` border + lavender-300 focus + `--shadow-focus` ring + `::placeholder` on-glass-subtle; secondary buttons = `--glass-bg` + `--glass-border-strong`; danger text/hover = `#ffb4b4` / `rgba(214,69,69,0.24)`; all-clear success text = `#6ee7b7`; attention/warning text = `#f5c97a` (aligned to Maintenance's moderate amber).
- **Status "colored dot + label" chip in a CSS Module (no JSX change):** the `.badge*`/`.pill*` rules use `display:inline-flex; gap` + a `::before { content:''; width/height:6px; border-radius:pill; background: currentColor }` — the dot inherits the label's brightened semantic color automatically.
- **Properties GLASSED its overlay modal forms** (Edit Property + New Property modals: `--glass-bg-strong` panel + blur + `--glass-border-strong`, over the existing `--color-overlay` ink scrim). This is now the universal pattern (see the RESOLVED glass-modals note below) — Maintenance was migrated to match it.
- **Dead files LEFT AS-IS (opposite of the Conversations pass's choice):** the live detail view is the **self-contained** `PropertyProfile.tsx` (own inline `ProfileCard`/`EditPropertyModal`/`UnitList`/`UnitCard`/`TenantRow` + `PropertyProfile.module.css`). Siblings `ProfileCard.tsx`, `UnitList.tsx`, `EditPropertyModal.tsx`, `propertyStyles.ts`, `ticketStatus.ts` are **orphaned** (imported by nothing on any page). They lean on Tailwind token utilities (`bg-bg-surface`, `text-ink`, `rounded-pill`) that need a `@theme` block — `propertyStyles.ts` even comments one exists, but **no `@theme` block exists anywhere in the repo**, so those utilities emit no CSS. I did NOT restyle them (would be misleading busywork on unreachable code); flagged to the user to delete or wire up. `PropertyNav.tsx` left light on purpose — it lives in the shared `SidebarTabs` panel next to the still-relevant light chrome; treat as sidebar chrome.

**RESOLVED — GLASS MODALS EVERYWHERE, including Maintenance (2026-07-22).** The user decided all overlay forms across all pages get the glass treatment. Maintenance's `TicketModal`/`NewTicketModal` (`components/maintenance/TicketList.tsx`) and the mobile filter bottom-sheet (`MobileTicketList.tsx`) were migrated from light `--color-bg-surface` to the same glass-modal pattern Properties/Conversations use: **hand-rolled glass panel over the existing `--color-overlay` ink scrim** — panel = `border [border-color:var(--glass-border-strong)] [background:var(--glass-bg-strong)] shadow-[var(--glass-shadow)] backdrop-blur-[var(--glass-blur)]`. Hand-rolled (NOT `.glass-panel`) because `.glass-panel` hard-sets `border-radius: var(--radius-lg)` and the cascade gotcha means Tailwind can't override it — modals need `rounded-t-[var(--radius-xl)]` (bottom-sheet) / `md:rounded-[var(--radius-xl)]`. Inside: `.glass-chip` for the grouped editable box + status badge, `glassInputClass`/`glassSelectClass`/`glassTextareaClass` for fields, `glassErrorClass` for error banners, on-glass text tokens, glass ghost Cancel (`hover:bg-white/10`), and the lavender primary (below). `PriorityBadge` now renders `onGlass` in the detail modal too (its light branch is now unused-but-kept for reusability).
- **How to apply:** no more "ask if overlay forms stay light" — glass every modal/drawer/sheet. Reuse the pattern above verbatim.

**RESOLVED — bright lavender is THE primary CTA everywhere (2026-07-22).** All pages now use the **bright `--color-lavender-300` pill w/ `--color-ink` text** (`hover:[background:var(--color-lavender-200)]` + `shadow-[var(--glass-shadow)]`) as the primary action on glass. Maintenance's earlier solid-`--color-ink`-with-white-text buttons ("New Ticket" desktop + mobile FAB, "Save changes", "Create ticket", filter-sheet "Done") were all recolored to lavender. The user picked lavender over the dark-ink pill because Settings/Conversations/Properties had independently converged on it. Default to lavender primary on any new glass surface; do NOT use a solid `--color-ink` fill for a primary button (it blends into the gradient's dark corner).

**Form-control-on-glass conventions (established pass 2 / Settings — Overview had no forms):** reuse these verbatim so the remaining pages' forms match.
- **Primary CTA button:** solid `[background:var(--color-lavender-300)] [color:var(--color-ink)]` + `hover:[background:var(--color-lavender-200)]` + `shadow-[var(--glass-shadow)]`. Bright lavender pill = the primary action on the dark gradient (a `--color-ink` fill would blend into the gradient's dark corner). Used for Save Changes / mobile Save / push Enable.
- **Secondary button:** `[background:var(--glass-bg)] [border-color:var(--glass-border-strong)] [color:var(--color-on-glass)] hover:[background:var(--glass-bg-strong)]` (hand-rolled, NOT `.glass-chip`, so the `hover:` utility actually applies — see cascade gotcha).
- **Text input / select / textarea:** `[background:var(--glass-bg-strong)] [border-color:var(--glass-border-strong)] [color:var(--color-on-glass)] placeholder:[color:var(--color-on-glass-subtle)] focus:[border-color:var(--color-lavender-300)] focus:shadow-[var(--shadow-focus)]`. `--shadow-focus` (existing lavender glow token) doubles as the on-glass focus ring — reuse it, don't invent one. Keep `text-base` (≥1rem anti-zoom rule).
- **`<select>` gotcha:** the dropdown popup inherits the control's white text → white-on-white/invisible options in some browsers. Pin option colors: `[&>option]:[background:var(--color-ink)] [&>option]:[color:var(--color-on-glass)]`.
- **Toggle switch (peer-checkbox pattern):** off-track `[background:rgba(255,255,255,0.22)]`, on-track `peer-checked:[background:var(--color-lavender-300)]`, white knob, focus `peer-focus-visible:ring-2 peer-focus-visible:ring-[var(--color-lavender-300)]` (dropped the `ring-offset-2` — a white offset ring looks wrong on dark).
- **Checkbox "chip" (label wrapping a native checkbox):** unchecked = `[border-color:var(--glass-border)] [color:var(--color-on-glass)]`; checked repaints the whole pill bright: `has-[:checked]:[background:var(--color-lavender-300)] has-[:checked]:[border-color:var(--color-lavender-300)] has-[:checked]:[color:var(--color-ink)]`, with `accent-[var(--color-ink)]` on the checkbox so its check reads on the lavender pill.
- **Danger/success feedback banners & danger text:** no light `--color-danger-bg`/`--color-success-bg` on glass. Danger text/icons = `#ffb4b4`; danger surface = `[background:rgba(214,69,69,0.16)] [border-color:rgba(255,180,180,0.4)]`. Success = text `#6ee7b7`, surface `[background:rgba(30,158,108,0.16)] [border-color:rgba(110,231,183,0.38)]`. (Inline arbitrary values, NOT new tokens — same approach as Overview's error card.)
- **Avatars / join-code / read-only chips:** `.glass-chip [color:var(--color-on-glass)]`. RoleBadge admin = bright lavender pill; member = `.glass-chip`.
- **Full-screen mobile drill-down (`MobileSettingsPanel`):** paints `--gradient-app-bg` on its own `fixed inset-0` container; frosted top bar = `[background:var(--glass-bg)] backdrop-blur-[var(--glass-blur)]` + `--glass-border` bottom border. Panel content renders on the gradient directly on mobile (no inner `.glass-panel` wrapper) but inside the desktop `.glass-panel` tabpanel — `.glass-chip` children read fine on both.
- **Reusable pill controls (from the Maintenance pass):** `components/maintenance/TicketList.tsx` EXPORTS `glassInputClass`, `glassSelectClass`, `glassErrorClass` implementing the above conventions as `rounded-full` frosted pills (`backdrop-blur-md`, glass-bg-strong, lavender `--shadow-focus` ring, dark `--color-ink` `<option>` popup). Also a module-local `glassTextareaClass` = same tokens but `rounded-[var(--radius-md)]` (a pill textarea is unusable). These are used by the toolbar AND (as of 2026-07-22) the glass modal forms — the inputs/selects render as full-width pills inside the modals, which reads fine and stays consistent with the toolbar. `MobileTicketList.tsx` imports `glassInputClass`, `glassSelectClass`, `glassErrorClass`. Import these for another page's search/filter/form instead of re-typing the arbitrary-value soup.
- **Badge shared between glass + light surfaces → add an `onGlass` prop, don't fork the component.** Maintenance `PriorityBadge` renders on both the glass board card and the light detail modal: `onGlass` toggles `.glass-chip` + brightened text (mild `#7ee7b0` / moderate `#f5c97a` / severe `#ffb4b4`) vs sunken fill + saturated `--color-success`/`--color-warning`/`--color-danger` text. Severity hue language = green Mild / amber Moderate / red Severe (dot uses the saturated token on both surfaces).

**New additive tokens in `app/globals.css` (`:root`)** — the old lavender/ink tokens are untouched:
- `--gradient-app-bg` (135deg deep-indigo `#241c54` → violet `#3b2e7a` → blue-purple `#5f4fa0`; white text clears ~6.7:1 even on the lightest stop).
- Glass: `--glass-bg`, `--glass-bg-strong`, `--glass-border`, `--glass-border-strong`, `--glass-blur` (20px), `--glass-shadow`.
- Text on glass: `--color-on-glass` (white), `--color-on-glass-muted`, `--color-on-glass-subtle`.
- Chart-on-glass (used via inline SVG `style`, per the SVG-var rule in [[nextjs-quirks]]): `--chart-line-primary` (white), `--chart-line-secondary` (`#b7a6ff` lavender), `--chart-line-muted`, `--chart-grid`, `--chart-axis-text`.

**Reusable utility classes (plain CSS in globals.css, not @layer):** `.glass-panel` (frosted fill + hairline border + shadow + `backdrop-filter: blur` + `--radius-lg`), `.glass-chip` (more-opaque fill + border, NO backdrop-filter — meant to sit ON a glass-panel), `.glass-interactive` (transition + `:hover` brighten fill/border; add to a clickable `.glass-panel`).

**CASCADE GOTCHA (important):** `.glass-panel`/`.glass-chip`/`.glass-interactive` are **unlayered** plain CSS, and unlayered rules **beat Tailwind's `@layer utilities`**. So a Tailwind `hover:bg-white/10` (or any `[background:…]`) utility **cannot override** a `.glass-panel` background/border. That's exactly why `.glass-interactive:hover` exists (its `:hover` specificity + unlayered status wins). When you need to vary a glass surface's fill, either use a different unlayered rule or hand-roll the surface with Tailwind arbitrary values instead of `.glass-panel` (as the Overview error card does).

**Shell/background wiring (`app/dashboard/dashboard.module.css`):**
- `.shell` now has `background: var(--gradient-app-bg)` (shell-level, ready for all pages).
- `.main` still paints `var(--color-bg-base)` (light) ON PURPOSE — it covers the shell gradient so un-migrated pages stay light. A **migrated page reveals the gradient by painting `--gradient-app-bg` on its own scroll container** (Overview's root `overflow-y-auto` div does this; default `background-attachment` keeps it fixed while content scrolls). **End state:** once every page is migrated, delete `.main`'s background and the shell gradient shows through globally.

**Sidebar/tab bar decision:** `IconSidebar` + `MobileTabBar` were left as **solid `--color-ink` chrome** (not glassed). Reason: they're shared across ALL dashboard pages via `--color-sidebar-bg`, so glassing them would wreck the still-light pages; and ink (`#2a225c`) already matches the gradient's dark corner, so the rail reads as a cohesive dark anchor beside the glass content. Revisit only once every page is migrated.

**Status chips follow the reference "colored dot + label" pattern:** e.g. escalated = `.glass-chip` pill with a `--color-danger` dot + light-red (`#ffb4b4`) text. Notification/unread counts use a solid `--color-lavender-300` pill with `--color-ink` text (bright, high-contrast on dark). Icon chips convey status by hue (lavender-accent / white / danger-red), not by card fill — every card is the same glass.
